TrainerMate Documentation

About TrainerMate
TrainerMate helps authorised trainers manage scheduled training sessions and connect Zoom meetings to those sessions. It integrates with Zoom using OAuth and uses Zoom meeting APIs to create, verify, and manage scheduled Zoom meetings.

1. Adding the App

To add TrainerMate to your Zoom account:

  1. Open TrainerMate.
  2. Go to the Zoom accounts section.
  3. Click Connect Zoom account.
  4. You will be redirected to Zoom to authorize TrainerMate.
  5. Review the requested permissions.
  6. Click Allow to approve access.
  7. After approval, Zoom redirects back to TrainerMate using the OAuth callback URL.
  8. Your Zoom account will appear as a connected account inside TrainerMate.

Production OAuth callback URL:

https://www.trainermate.xyz/zoom/callback

2. Using the App

2.1 Connect a Zoom Account

Users connect Zoom through the TrainerMate dashboard. TrainerMate never asks for or stores a Zoom password. Authorization is completed through Zoom OAuth.

2.2 Verify Existing Zoom Meetings

TrainerMate can check whether a scheduled training session already has a valid Zoom meeting. Where meeting details already exist, TrainerMate verifies the exact Zoom Meeting ID against the connected Zoom account.

2.3 Create Zoom Meetings

If a scheduled training session does not already have a valid Zoom meeting, TrainerMate can create a scheduled Zoom meeting using the connected Zoom account.

The created meeting may include:

  • Meeting topic/title
  • Scheduled start time
  • Meeting duration
  • Join URL
  • Meeting ID
  • Meeting password

2.4 Update or Replace Meeting Details

TrainerMate avoids unnecessary duplicate meetings. If existing meeting details are still live in Zoom, they are reused. If meeting details are stale or no longer valid, TrainerMate can prepare replacement meeting details.

2.5 Multiple Zoom Accounts

TrainerMate supports linked Zoom account selection so different providers or workflows can use different connected Zoom accounts where appropriate.

2.6 Provider-Managed Meetings

Some providers manage their own Zoom meeting details. In these cases, TrainerMate can be configured as read-only for that provider. Read-only mode prevents Zoom writes while still allowing permitted verification behavior.

3. Removing the App

You can remove TrainerMate access from your Zoom account at any time.

3.1 Remove from TrainerMate

  1. Open TrainerMate.
  2. Go to the Zoom accounts section.
  3. Select the connected Zoom account.
  4. Click Disconnect.
  5. The account is removed from TrainerMate's connected Zoom account list.

3.2 Remove from Zoom Marketplace

  1. Sign in to your Zoom account.
  2. Open the Zoom App Marketplace.
  3. Go to Manage.
  4. Find TrainerMate in your added apps.
  5. Click Remove or Uninstall.
  6. Confirm removal.
Effect of removing the app:
Removing TrainerMate revokes its ability to access your Zoom account through OAuth. TrainerMate will no longer be able to create, read, verify, or update Zoom meetings for that connected account unless you reconnect it.

4. Data Handling and Deauthorization

TrainerMate uses Zoom data only to provide Zoom meeting management functionality.

4.1 Data Stored

TrainerMate may store the following Zoom meeting metadata locally:

  • Meeting ID
  • Join URL
  • Meeting password
  • Meeting topic/title
  • Scheduled start time

4.2 Token Storage

Zoom OAuth access and refresh tokens are stored using secure local credential storage, such as the operating system's keyring or credential manager.

4.3 Data Not Stored

  • TrainerMate does not collect or store Zoom account passwords.
  • TrainerMate does not sell Zoom data.
  • TrainerMate does not share Zoom data with third parties.

4.4 Data Removal After Deauthorization

When a user disconnects or removes TrainerMate, OAuth access is revoked. Stored OAuth tokens are removed from local credential storage where applicable. Local meeting metadata may remain on the user's device for audit or session history unless the user deletes the local TrainerMate data.

5. Troubleshooting

5.1 Zoom Authorization Fails

  • Confirm you are using the correct Zoom account.
  • Confirm popup blockers or browser security tools are not blocking the OAuth page.
  • Try reconnecting the Zoom account from TrainerMate.

5.2 Callback Page Shows Missing Details

If the callback page says Zoom did not return the required details, restart the Zoom connection flow from TrainerMate. The callback URL should normally be reached after Zoom returns an authorization code.

5.3 Meeting Verification Fails

  • Reconnect the Zoom account.
  • Confirm the connected Zoom account owns or can access the meeting being verified.
  • Confirm the meeting has not been deleted in Zoom.

5.4 Duplicate Meetings

TrainerMate verifies exact Meeting IDs before creating replacements. If duplicate meetings are suspected, check the connected Zoom account and reconnect Zoom if needed.

5.5 Provider Portal Access

Provider portal access is controlled by the relevant third-party provider. TrainerMate users must only use credentials they are authorized to use.

6. Zoom Scopes Used

TrainerMate requests Zoom permissions only for the Zoom meeting functionality it provides.

  • Read meeting information to verify existing scheduled meetings.
  • Create scheduled meetings when required for training sessions.
  • Update meeting details when necessary.
