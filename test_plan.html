1. Product Overview

TrainerMate helps authorised trainers manage scheduled training courses and connect those courses to Zoom meetings. A trainer signs in to TrainerMate, reviews scheduled courses, connects their Zoom account, and uses TrainerMate to create, verify, or refresh the Zoom meeting associated with each course.

The live product can use local trainer workflows and provider-managed course sources. For this Marketplace review, the hosted environment uses realistic seeded course records so Zoom reviewers can test the full Zoom workflow without private third-party provider credentials.

3. Preconditions Before Testing

  1. Open https://demo.trainermate.xyz.
  2. Sign in using reviewer@zoom.us and the password supplied in the Marketplace submission notes.
  3. Use a Zoom account suitable for testing app authorization and meeting creation.
  4. Confirm the browser remains on the demo.trainermate.xyz domain for TrainerMate pages.
Do not use the Render temporary URL for OAuth testing. Zoom OAuth should be tested from https://demo.trainermate.xyz because the approved callback is https://demo.trainermate.xyz/zoom/callback.

4. Dashboard Login and Course Visibility

  1. Go to https://demo.trainermate.xyz.
  2. Enter the reviewer email and password.
  3. Click sign in.
  4. Confirm the TrainerMate dashboard opens.
  5. Review the visible scheduled courses in the dashboard.
Expected result: The reviewer can access the TrainerMate dashboard and see realistic scheduled training courses with status indicators such as synced/not checked.

5. Behaviour Before Zoom Is Connected

  1. Ensure no Zoom account is connected in TrainerMate.
  2. From the dashboard, attempt to check or sync a course.
Expected result: TrainerMate shows a friendly message explaining that Zoom must be connected before courses can be synced. It should not show a raw token, OAuth, server, or traceback error.

6. Zoom OAuth Authorization

  1. Open Zoom accounts from the TrainerMate navigation.
  2. Click Connect Zoom account.
  3. Review the Zoom authorization screen.
  4. Approve the TrainerMate app.
  5. Confirm Zoom redirects back to https://demo.trainermate.xyz/zoom/callback.
Expected result: OAuth authorization completes using the Production Client ID. TrainerMate exchanges the authorization code, stores the Zoom connection, and returns the reviewer to the TrainerMate dashboard.

7. Zoom Account Connection Verification

  1. After OAuth returns to TrainerMate, open Zoom accounts.
  2. Confirm the connected Zoom account is visible.
  3. Confirm the dashboard no longer prompts to connect Zoom before syncing.
Expected result: The connected Zoom account appears in TrainerMate. This validates the user/account read permission and successful token exchange.

8. Existing Meeting Read / Verification

  1. Return to the dashboard.
  2. Select one of the courses already showing as synced, if available.
  3. Run the course check/sync action.
Expected result: TrainerMate uses Zoom meeting read access to verify the saved meeting state and displays a clear result in the dashboard.

9. Meeting Creation / Course Sync

  1. Choose a course that is not yet checked or synced.
  2. Click the available check/sync action.
  3. Wait for TrainerMate to create or sync the Zoom meeting for the course.
  4. Confirm the course status updates after completion.
Expected result: TrainerMate creates a Zoom meeting using the course title, date, and time, then updates the course row with the synced meeting state.

10. Meeting Replacement / Update Behaviour

  1. Use a course that already has a synced Zoom meeting.
  2. Run the check/sync action again.
  3. If the existing meeting is present, TrainerMate should verify it.
  4. If the existing meeting is missing or invalid, TrainerMate should refresh or recreate the meeting and update the dashboard state.
Expected result: TrainerMate handles existing and missing Zoom meeting states gracefully, without exposing technical errors to the trainer.

11. Disconnect / Reconnect Behaviour

  1. Open Zoom accounts.
  2. Click the disconnect option for the connected Zoom account.
  3. Confirm the Zoom account is no longer shown as connected.
  4. Attempt a course sync and confirm TrainerMate asks for Zoom to be connected.
  5. Click Connect Zoom account again and complete OAuth to reconnect.
Expected result: TrainerMate disconnects the saved Zoom connection, blocks sync with a clear friendly message, and allows the trainer to reconnect through Zoom OAuth.

12. Expected End-User Experience

  • The trainer signs in to TrainerMate.
  • The trainer sees scheduled courses in the dashboard.
  • The trainer connects Zoom once through Zoom OAuth.
  • TrainerMate can create, verify, refresh, or reconnect meetings for scheduled courses.
  • If Zoom is not connected or needs attention, TrainerMate shows a clear non-technical message and directs the trainer to reconnect Zoom.

Notes on Provider Credentials

The hosted review environment does not expose private third-party provider portals or provider credentials. Instead, it uses realistic seeded course data to allow reviewers to test the complete TrainerMate Zoom workflow safely. The Zoom authorization, token exchange, account verification, meeting read, meeting create, and disconnect/reconnect behaviour are real and use the Production Client ID.
