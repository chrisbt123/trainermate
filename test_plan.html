TrainerMate Zoom Marketplace Review Test Plan

English step-by-step review guide for Zoom Marketplace reviewers.

Reviewer environment:
URL: https://demo.trainermate.xyz
Login email: reviewer@zoom.us
Password: supplied in the Zoom Marketplace submission notes.
OAuth redirect URI: https://demo.trainermate.xyz/zoom/callback
The environment uses the Production Client ID during authorization.
Provider credentials note: TrainerMate is used by authorised trainers with private third-party provider systems. Real provider/FOBS credentials cannot be shared externally. This review environment uses realistic seeded course records so Zoom reviewers can test the customer-facing TrainerMate workflow and all Zoom functionality without accessing private third-party systems.

1. Product overview

TrainerMate helps trainers manage scheduled course meetings. A trainer connects their Zoom account, then TrainerMate creates, verifies, updates, or replaces the Zoom meetings associated with scheduled training courses.

The reviewer environment shows the TrainerMate dashboard with seeded course data. The course data is seeded, but the Zoom OAuth flow and Zoom API calls are real.

3. Full review walkthrough

Step 1 — Log in to TrainerMate

  1. Open https://demo.trainermate.xyz.
  2. Enter reviewer@zoom.us.
  3. Enter the password supplied in the Marketplace submission notes.
  4. Click sign in.

Expected result: The TrainerMate dashboard opens with scheduled course data visible. The account is configured as a paid/full review account so the reviewer can test the full sync window.

Step 2 — Review the scheduled courses

  1. On the dashboard, review the Next courses list and the course status labels.
  2. Some courses may already show as synced; other courses will show as not checked.

Expected result: The reviewer can see realistic scheduled course records. This simulates the trainer's normal course list without requiring private provider portal credentials.

Step 3 — Confirm gentle handling before Zoom is connected

  1. Before connecting Zoom, attempt to run a course check/sync or the dashboard sync action.

Expected result: TrainerMate should not start a sync without Zoom. It should show a plain-English message explaining that Zoom must be connected before syncing courses.

Step 4 — Connect Zoom

  1. Open Zoom accounts.
  2. Click Connect Zoom account.
  3. Approve the TrainerMate app in Zoom.
  4. Confirm Zoom redirects back to https://demo.trainermate.xyz/zoom/callback.

Expected result: TrainerMate returns to the dashboard and displays the connected Zoom account. Authorization uses the Production Client ID.

Step 5 — Verify an existing synced course meeting

  1. Return to the dashboard.
  2. Choose a course that already shows as synced.
  3. Run the course check/sync action.

Expected result: TrainerMate checks the saved Zoom meeting through the Zoom Meeting API. If the meeting exists, TrainerMate verifies and refreshes/updates that same meeting record. It should not create a duplicate meeting. The course row should remain synced and the Notes column should show a re-verification message, for example Zoom re-verified - FOBS + Zoom OK.

Step 6 — Create a meeting for an unchecked course

  1. Choose a course marked Not checked.
  2. Run the course check/sync action.

Expected result: TrainerMate creates a Zoom meeting for that scheduled course and updates the course status.

Step 7 — Confirm update behaviour on the same course

  1. Run the check/sync action again on the course that was just synced.

Expected result: TrainerMate verifies and updates the same Zoom meeting rather than creating a duplicate. The course Notes column should show that the existing Zoom meeting was re-verified/refreshed.

Step 7a — Confirm visible course note after re-verification

  1. After running check/sync on an already-synced course, look at the course row in the dashboard.
  2. Review the right-side Notes column.

Expected result: The course remains synced and the Notes column displays a clear message such as Zoom re-verified - FOBS + Zoom OK. This confirms the existing meeting was checked/refreshed rather than duplicated.

Step 8 — External deletion / replacement behaviour

  1. Optional: delete a TrainerMate-created meeting directly from Zoom.
  2. Return to TrainerMate and run check/sync on the affected course.

Expected result: TrainerMate detects that the saved meeting is no longer available. It attempts to locate an existing matching course meeting first. If no valid existing meeting can be found, TrainerMate creates a replacement meeting.

Step 9 — Disconnect and reconnect Zoom

  1. Open Zoom accounts.
  2. Click Disconnect.
  3. Confirm the Zoom account is removed.
  4. Click Connect Zoom account again to repeat the OAuth flow if required.

Expected result: TrainerMate disconnects the Zoom account and can reconnect using the same Production OAuth flow.

4. Expected user-facing experience

  • Users should see plain-English connection states such as Zoom connected, Zoom is not connected yet, or Zoom needs a quick refresh.
  • Users should not need to understand OAuth tokens, callback state, or API errors.
  • TrainerMate should verify/update existing course meetings before creating replacements.
  • After an existing meeting is verified/refreshed, the course Notes column should show a visible re-verification status.
  • TrainerMate should create a new meeting only when no valid existing meeting is linked or found.
