2. Product overview

TrainerMate helps authorised trainers manage scheduled training courses and associated Zoom meetings. A trainer connects their Zoom account, then TrainerMate checks scheduled courses, verifies whether a Zoom meeting already exists for each course, and creates or refreshes meetings where required.

The live product also interacts with private provider systems used by trainers. Those provider portals are third-party protected systems and credentials cannot be shared externally. For Marketplace review, this hosted environment uses seeded course records while preserving the customer-facing TrainerMate dashboard and Zoom workflow.

4. Scope description

TrainerMate integrates with the Zoom API to create, read, and manage Zoom meeting data on behalf of an authorized Zoom user.

The application uses Zoom data to:

  • Confirm the connected Zoom user account after OAuth authorization.
  • Retrieve the user's meetings to determine whether a scheduled training course already has a corresponding Zoom meeting.
  • Read a specific meeting by meeting ID to verify whether the meeting currently linked to a course still exists.
  • Create a new scheduled Zoom meeting when no valid existing meeting is linked or found for a course.
  • Refresh or update the local course meeting state where applicable so the course remains aligned with the associated Zoom meeting.
  • Prevent duplicate meeting creation by verifying existing linked meetings and searching for matching meetings before creating a new one.

Zoom access tokens and refresh tokens are stored securely using the operating system's credential manager/keyring where the local desktop version is used, or secured environment/runtime storage where the hosted reviewer environment is used. Tokens are never exposed to end users and are not transmitted to unrelated external services.

Meeting metadata returned by the Zoom API, including meeting ID, join URL, topic, start time, and meeting password where applicable, may be stored locally or in the application's runtime data store. This data is used solely to associate Zoom meetings with scheduled training courses and to verify or refresh those meetings later.

No Zoom account passwords are collected, processed, or stored by TrainerMate.

TrainerMate does not share Zoom data with third parties. All Zoom data access is limited to the authenticated user's Zoom account and is used strictly for the course meeting functionality described in this test plan.

5. Step-by-step test flow

Step 1 — Log in to TrainerMate

  1. Open https://demo.trainermate.xyz.
  2. Sign in using the reviewer email and password supplied in the submission notes.
  3. Confirm the TrainerMate dashboard loads with seeded scheduled courses.

Expected result: The dashboard loads, the account is available for review, and scheduled courses are visible.

Step 2 — Confirm sync is blocked before Zoom is connected

  1. Before connecting Zoom, click a course sync/check action or click Sync quietly.

Expected result: TrainerMate should not start syncing. It should display a friendly message similar to: Zoom is not connected yet. Open Zoom accounts and connect Zoom before syncing courses.

Step 3 — Connect Zoom

  1. Open Zoom accounts.
  2. Click Connect Zoom account.
  3. Approve TrainerMate in the Zoom authorization screen.
  4. Confirm Zoom redirects back to https://demo.trainermate.xyz/zoom/callback.

Expected result: TrainerMate displays the connected Zoom account. This demonstrates OAuth authorization, redirect handling, token exchange, and user:read:user account verification.

Step 4 — Verify an existing synced course meeting

  1. Open the dashboard.
  2. Select or run check/sync on a course that already appears as synced.
  3. Observe the status/notes for that course.

Expected result: TrainerMate verifies the existing linked Zoom meeting using the saved meeting ID. The course should remain synced and the notes/status may show wording such as Zoom re-verified. No duplicate meeting should be created.

Step 5 — Create a meeting for an unchecked course

  1. Select a course that is shown as not checked or not yet synced.
  2. Run the course check/sync action.

Expected result: TrainerMate creates one scheduled Zoom meeting for the selected course and links the meeting ID/metadata to the course.

Step 6 — Re-run check/sync on the same course

  1. Run the same course check/sync action again on the course from Step 5.

Expected result: TrainerMate verifies or refreshes the same existing meeting rather than creating a duplicate meeting. This demonstrates the meeting read/list behaviour and the meeting write capability where a course meeting needs to be refreshed.

Step 7 — Externally deleted meeting scenario

  1. If a meeting has been deleted directly in Zoom, run the course check/sync action again in TrainerMate.

Expected result: TrainerMate detects the missing linked meeting, searches for a matching course meeting, and creates a replacement only if no valid existing meeting can be found.

Step 8 — Disconnect and reconnect Zoom

  1. Open Zoom accounts.
  2. Click Disconnect.
  3. Confirm the Zoom account is removed from TrainerMate.
  4. Click Connect Zoom account again to confirm reconnect works.

Expected result: TrainerMate disconnects and reconnects cleanly. The user is sent through the Zoom authorization flow again when reconnecting.

6. End-user visible functionality

  • TrainerMate dashboard login.
  • Scheduled course list and course status.
  • Zoom account connection and disconnection.
  • Friendly message when syncing is attempted before Zoom is connected.
  • Course meeting verification for already-synced courses.
  • Meeting creation for courses without a valid linked meeting.
  • Meeting refresh/update behaviour for existing course meetings.
  • Duplicate prevention by verifying existing meetings before creating a new meeting.

7. Notes on provider credentials

The live TrainerMate workflow may use private third-party provider portals that are only available to authorised trainers. Those credentials cannot be shared externally. The review environment therefore uses realistic seeded course records so Zoom reviewers can test the complete Zoom-facing TrainerMate workflow without requiring access to those private provider systems.

8. Expected reviewer completion criteria

  • Reviewer can log in to TrainerMate.
  • Reviewer can authorize Zoom using the Production Client ID.
  • Reviewer can see the connected Zoom account after callback.
  • Reviewer can verify an existing linked meeting.
  • Reviewer can create a meeting for an unchecked course.
  • Reviewer can run the same action again without creating a duplicate meeting.
  • Reviewer can disconnect and reconnect Zoom.
