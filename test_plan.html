TrainerMate Zoom Marketplace Reviewer Test Plan

Reviewer Note:

TrainerMate is currently delivered as a secure local desktop workflow. Zoom reviewers do not need access to any FOBS or third-party provider portals, and no provider credentials are supplied as these are private third-party systems. Provider access is not part of Zoom Marketplace review. Review should focus only on the Zoom OAuth authorization flow, callback handling, requested Zoom scopes, and Zoom meeting read/create/update functionality.

1. App Overview

TrainerMate is a training session management platform that integrates with Zoom to automate Zoom meeting creation, verification, replacement, and synchronization for scheduled training courses.

The Zoom integration allows users to:

  • Connect their Zoom account securely using Zoom OAuth
  • Read existing scheduled Zoom meetings
  • Create new Zoom meetings when required
  • Update or replace stale Zoom meetings
  • Store meeting metadata locally for course association
  • Disconnect Zoom authorization safely

2. Adding and Authorizing the App

  1. Open TrainerMate dashboard
  2. Navigate to the Zoom Account Linking section
  3. Click Connect Zoom Account
  4. You will be redirected to Zoom OAuth authorization
  5. Approve the requested permissions using your Zoom account
  6. After approval, Zoom redirects back to:
    https://www.trainermate.xyz/zoom/callback
  7. The linked Zoom account will now appear inside TrainerMate

3. Testing Meeting Read Access

  1. With a connected Zoom account, TrainerMate checks scheduled meetings
  2. The system reads Zoom meetings using the authorized Zoom account
  3. Existing meetings are matched using exact Zoom Meeting ID verification
  4. Live meetings are confirmed and reused instead of duplicated

4. Testing Meeting Creation

  1. If no valid existing Zoom meeting is found, TrainerMate creates a new scheduled Zoom meeting
  2. The meeting includes title, scheduled date/time, meeting link, and password
  3. The returned Zoom meeting metadata is stored locally for future verification

5. Testing Meeting Update / Replacement

  1. If stale or invalid Zoom details are detected, TrainerMate prepares a replacement
  2. Only invalid or non-live meetings are replaced
  3. Live meetings are preserved and synced instead of overwritten

6. Disconnecting / Removing the App

  1. Users may disconnect Zoom authorization at any time
  2. OAuth tokens are invalidated by deauthorization
  3. No Zoom passwords are ever stored
  4. Stored meeting metadata remains local to the user environment only

7. Data Handling

  • Zoom access and refresh tokens are stored securely using encrypted system credential storage
  • Meeting metadata (meeting ID, join URL, password) may be stored locally in SQLite
  • No Zoom passwords are collected or stored
  • No Zoom data is sold or shared with third parties

8. Reviewer Credentials

No FOBS or provider portal credentials are provided because these are private third-party credentials and are not required for Zoom Marketplace review.

Reviewers should validate the Zoom OAuth authorization flow and Zoom API functionality using their own Zoom account through the production OAuth authorization flow.
