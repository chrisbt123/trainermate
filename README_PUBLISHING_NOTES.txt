TrainerMate website - GitHub Pages ready build

How to publish:
1. Upload/replace the contents of this zip at the GitHub Pages publishing root.
2. Do not upload the enclosing folder itself. index.html, CNAME, .nojekyll and assets/ should sit directly in the publishing root.
3. Commit the changes and wait for GitHub Pages to deploy.
4. Open https://www.trainermate.xyz/. Returning visitors receive the current
   cache-busted assets and the site refresh helper automatically.

This build includes:
- .nojekyll for GitHub Pages static publishing.
- CNAME for www.trainermate.xyz.
- assets/styles.css plus inline CSS fallback inside each page.
- assets/site-refresh.js to refresh returning visitors after a site release.
- site-version.json so an older page can notice a newer site release.
- release.json so installed apps can gently offer the latest verified installer.
- Sales-focused copy that pushes TrainerMate automation and time saving.
