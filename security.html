Security approach

TrainerMate is designed as a local-first desktop assistant with minimal remote access and cautious automation.

Local-firstTrainer-owned provider data, course cache and documents are intended to stay on the trainer’s computer unless the trainer chooses to send support information.
CredentialsProvider credentials and Zoom secrets should be stored using the operating system credential store where available, not hard-coded into plain text files.
Zoom OAuthTrainerMate uses Zoom OAuth authorisation. Users authorise the app with Zoom; TrainerMate should not ask for or store the trainer’s Zoom password.
Support packsSupport information should be redacted before sending and should avoid provider passwords, Zoom access tokens, refresh tokens, licence keys and unnecessary personal data.
Local accessThe local dashboard and admin API should bind to 127.0.0.1 and reject non-local requests.
XSS protectionPages should escape user/provider/support content and use defensive response headers including Content Security Policy where possible.
UpdatesUpdate downloads should use HTTPS. Automatic launch should only happen after a configured SHA-256 checksum matches.
Responsible use: Automation can reduce admin time, but trainers remain responsible for checking course details, Zoom links and provider portal information manually before relying on them.
